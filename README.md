## 设计思路

### 基本功能要求

#### 顾客功能

1. 用户注册：包括发送邮件确认功能。
2. 用户登录：允许用户访问他们的个人账户。
3. 购买流程：用户可以浏览商品，将商品添加至购物篮，进行付款，并确认发货。

#### 管理员功能

1. 商品目录管理：管理员可以添加、删除和修改商品信息。
2. 后台订单销售状态管理：包括取消订单和确认发货等操作。
3. 用户订单记录：管理员可以查看用户的订单历史记录。
